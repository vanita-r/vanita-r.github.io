Website: Andy Tran's Introduction Page

Website Checklist Evaluation

  • 1. Submission Link: Does the link lead directly to the page to be reviewed? No
  • 2. File/Folder Names: Are there no spaces or uppercase letters in file/folder names (including scripts, images, etc.)? Yes
  • 3. Design:
    • a. Page has sufficient contrast and font sizing for readability. No, there are 7 contrast errors in both the footer and header nav links.
    • b. Page uses site colors/fonts via a standard .css file. Yes
    • c. CRAP Principles:
      • i. Contrast - No. The black text on the pale yellow background is easy to read. However, the header and footer nav links both have 7 contrast errors.
      • ii. Repetition - Yes, the same colors and fonts are used throughout the site.
      • iii. Alignment - Yes, the elements are aligned properly. However, I believe it would look better if Andy's Name, avatar, and mascot were all center aligned.
      • iv. Proximity - Yes, related elements are grouped together.
  • 4. Page Structure: Does the page contain a Header, Main, and Footer? Yes
  • 5. Header: Is the brand/site header text in an <h1>? Yes
    • a. If this is a homework page it's likely something like Student Name * Course ID Course Name. No
    • b. If this is a "branded page" it should be the brand/company. Yes
    • c. DO NOT include page name in header. Yes
  • 6. Main Content: Does the main section start with the page name in an <h2>? (Should not include site/brand name). No
  • 7. Brand Tagline: Is there a consistent brand tagline or slogan on every page? Yes
  • 8. Footer:
    • a. Includes a menu for the user's pages. Yes
    • b. Includes HTML validation button/link. Yes
    • c. Includes CSS validation button/link. Yes

Constructive Feedback

  • The Good: The color scheme for the introduction page is consistent and visually appealing.
  • The Bad: It's hard to navigate through the site due to the lack of a clear menu structure.
  • The Ugly: The layout could be improved for better visual hierarchy and user experience. Also making the CLT webpage and GitHub more consistent helps so that it doesn't look like I'm going through 4 different websites.
